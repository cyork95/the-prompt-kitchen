🏆 Cook-Off

Every month, one prompt. Run through 5–10 AI tools. Results published here so you can compare what different models actually do with the same input. Try the prompt yourself and see where your results land.

How it works

1

One prompt

Each month we write a single prompt targeting a specific category — shopping, writing, coding, research, or creative tasks.

2

5+ tools tested

We run the exact same prompt through multiple AI tools, capturing raw unedited output from each one.

3

Results published

Full side-by-side results go live here. We note what worked, what didn't, and what surprised us.

4

You try it

Copy the prompt, run it yourself, and see how your results compare. Use the Test Kitchen to track your runs.

Or try the Test Kitchen — it has a curated library of challenge-style prompts across 6 categories, ready to run through any combination of models.

How we pick challenge prompts

A good challenge prompt has to work differently across tools — if every AI gives the same answer, it's not a useful comparison. We look for prompts that:

  • Have objective right/wrong elements — spec constraints, factual requirements, or format rules that can be clearly met or missed
  • Have subjective creative elements — so there's meaningful variation even when tools get the basics right
  • Stress-test common failure modes — things like following multi-part instructions, sticking to a budget, or applying specific constraints consistently
  • Are repeatable — you should be able to run the same prompt weeks later and still get a useful result

The shopping series was a perfect example: it had hard constraints (RTX 3060 or better, 16GB RAM min, $400–$800, specific store order) that many tools ignored or misread, plus enough complexity that the better tools could shine. Twenty-seven AIs, and still only a handful fully met the brief.

⚙️ New challenge pages are auto-generated

Each monthly challenge page is created automatically from a GitHub Actions workflow — no manual HTML editing required. The workflow takes a month name, category, and prompt as inputs and publishes a ready-to-fill challenge page at /challenges/{slug}/.

See the workflow: .github/workflows/new-challenge.yml
